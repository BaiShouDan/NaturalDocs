
Title: CSS Structure


Topic: Frame Page
_______________________________________________________________________________

(start text)

	<body NDPage NDFramePage IE6/7/8?>

		<#NDMessages>

			<#MsgCloseButton></#MsgCloseButton>

			<#MsgContent>
				<MsgMessage Debug?>
					message
				</MsgMessage>
				...
			</#MsgContent>

		</#NDMessages>

		<#NDHeader>
			<#HTitle>...</#HTitle>
			<#HSubTitle>...</#HSubTitle>
		</#NDHeader>

		<#NDLoadingNotice>
			(visible by default, removed by scripts once loaded)
		</#NDLoadingNotice>

		<#NDJavaScriptRequiredNotice>
			(only visible when JavaScript is disabled)
		</#NDJavaScriptRequiredNotice>

		<#NDMenu>...</#NDMenu>

		<#NDMenuSizer Dragging?>...</#NDMenuSizer>

		<#NDSummary>...</#NDSummary>

		<#NDSummarySizer Dragging?>...</#NDSummarySizer>

		<#NDContent>
			<iframe #CFrame></iframe>
		</#NDContent>

		<#NDFooter>
			<#FCopyright>...</#FCopyright>
			<#FTimeStamp>...</#FTimeStamp>
			<#FGeneratedBy>...</#FGeneratedBy>
		</#NDFooter>

	</body NDPage NDFramePage>

(end)

If the browser is Internet Explorer 6 through 8, the default style's JavaScript will add an IE6, IE7, or IE8 class to the body.



Topic: Menu
_______________________________________________________________________________

(start text)

	<#NDMenu>
		<#MContent>

			<MFolder MEntry Parent Root Selected?>
				Name
			</MFolder>

			<MFolder MEntry Parent Empty? Selected?>
				Name
			</MFolder>

			<MFile MEntry Selected>
				Name
			</MFile>

			<MFolder MEntry Child>
				<MFolderIcon></MFolderIcon>
				Name
			</MFolder>

			<MLoadingNotice>
			</MLoadingNotice>

		</#MContent>
	</#NDMenu>

(end)

	The menu always starts with a root folder entry.  There may also be a list of subfolder entries leading from it to the
	currently selected folder.  Most entries will be "a" tags, though they may be div tags if they don't need to serve as a
	link, such as empty and selected folders.  Use the Root, Parent, Child, and Selected classes to distinguish between types 
	of folder entries, such as .MFolder.Child.

	Following it will be a list of the files and folders contained in that location.

Topic: Prototype
_______________________________________________________________________________

No Parameter Form:

(start code)

	<NDPrototype NoParameterForm #NDPrototype[ID]>
		content
	</NDPrototype NoParameterForm #NDPrototype[ID]>

(end)


Short Form:

(start code)

	<NDPrototype ShortForm CStyle/PascalStyle #NDPrototype[ID]>
		<table>
			<tr>

				<td PBeforeParameters>
					...
				</td PBeforeParameters>

				<td PParametersParentCell>
					[Parameter Table]
				</td PParametersParentCell>

				<td PAfterParameters>
					...
				</td PAfterParameters>

			</tr>
		</table>
	</NDPrototype ShortForm CStyle/PascalStyle #NDPrototype[ID]>

(end)


Long Form:

(start code)

	<NDPrototype LongForm CStyle/PascalStyle #NDPrototype[ID]>
		<table>
			<tr>

				<td PBeforeParameters>
					...
				</td PBeforeParameters>

			</tr><tr>

				<td PParametersParentCell>
					[Parameter Table]
				</td PParametersParentCell>

			</tr><tr>

				<td PAfterParameters>
					...
				</td PAfterParameters>

			</tr>
		</table>
	</NDPrototype LongForm CStyle/PascalStyle #NDPrototype[ID]>

(end)


Parameter Table:

(start code)

		<table PParameters>
			<tr>
				<td PType>type<PTypeSuffix>suffix</PTypeSuffix></td PType>

				<td P[ColumnType]>column</td P[ColumnType]>
				<td P[ColumnType]>column</td P[ColumnType]>
				...
			</tr><tr>
				...
			</tr>
		</table PParameters>

(end code)

P[ColumnType]:

	PModifierQualifier - For C-style prototypes, a separate column for modifiers and qualifiers.
	PType - The parameter type.  For C-style prototypes this will only be the last word.  For Pascal-style
					prototypes this will be the entire symbol.
	PTypeNameSeparator - For Pascal-style prototypes, the symbol separating the name from the type.
	PNamePrefix - A prefix for a parameter name that should be formatted with the name, such as * and &.
	PName - The parameter name.
	PDefaultValueSeparator - If present, the symbol for assigning a default value like = or :=.
	PDefaultValue - The default value.

	<NDToolTip T[Type] L[Language]>

		<NDPrototype>...</NDPrototype>

The summary paragraph will not have separate <p> tags.
